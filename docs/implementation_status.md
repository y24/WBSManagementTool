# WBS管理ツール 実装ステータス確認リスト

現状（フェーズ7完了時点）での、要件定義に基づく実装状況をまとめました。

## 1. 共通・基盤
- [x] プロジェクト構成（FastAPI + React + PostgreSQL）の初期セットアップ
- [x] DB自動作成・マイグレーション・初期データ投入スクリプトの作成
- [x] Tailwind CSS v4 によるモダンなUI/UXデザインの適用
- [x] ダークモード対応（ベーススタイル対応済み、切り替え機能は未）

## 2. データベース / バックエンド (API)
- [x] **マスタテーブル実装**（ステータス、サブタスク種別、メンバー、祝日、システム設定）
- [x] **業務テーブル実装**（プロジェクト、タスク、サブタスク）
- [x] **CRUD API**（プロジェクト・タスク・サブタスクの作成・更新・論理削除）
- [x] **一括取得 API**（フロントエンド描画用の階層データ、マスタの一括取得）
- [x] **高度なロジック**（日付の親要素への自動積み上げ計算、重複チェック等）

## 3. フロントエンド (WBSツリー)
- [x] **3階層ツリー描画**（プロジェクト > タスク > サブタスク の階層表示）
- [x] **展開・折りたたみ機能**（プロジェクト単位、タスク単位での開閉）
- [x] **インライン編集（自動保存）**（名称、日付、進捗、ステータス、担当者の直接編集）
- [x] **行操作機能**（新規プロジェクト追加、タスク/サブタスク追加、論理削除）
- [ ] **ドラッグ＆ドロップによる並び替え**（DNDライブラリ未導入）
- [ ] **詳細モーダル**（各項目の詳細設定やチケットID、メモの編集などは簡易インラインのみ）

## 4. フロントエンド (ガントチャート)
- [x] **日付ヘッダーの描画**（日単位、土日色分け、今日線（Red Line）の表示）
- [x] **ガントバーの描画**（計画（予定）と実績の2段表示）
- [x] **スクロール同期**（左側のWBSツリーと右側のガント領域の縦スクロールの完全同期）
- [x] **境界リサイズ**（ツリーとガントの表示幅をドラッグで調整する境界線）
- [ ] **ガントバーのドラッグによる期間編集**（現在は数値入力のみ、マウスドラッグでの期間変更は未実装）
- [ ] **表示スケール切り替え**（現在は日単位固定、週・月単位への切り替えは未実装）

## 5. その他
- [x] **マスタ管理画面**（DB上のマスタ情報を一覧表示するプレビュー画面）
- [ ] **祝日・非稼働日の考慮**（データ構造はあるが、ガントチャートのマス目に反映させるロジックは未）
- [ ] **外部エクスポート/インポート**（CSV出力等の連携機能は未）

---

> [!TIP]
> **現在のステータス**: ツールとしての「表示」「入力」「保存」のコアサイクルは完成しており、Excel感覚でのデータ入力とガントバーの自動反映が可能な状態です。
>
> 次のステップとしては、**「日付の自動計算ロジック（子要素の日付が変わったら親も変わる）」**や、**「ガントバー自体をドラッグして期間を延ばす操作感の改善」**などが挙げられます。
