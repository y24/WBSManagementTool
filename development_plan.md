# Web版WBS管理ツール 実装計画

`docs`ディレクトリの要件定義書（全体要件、UI要件、DB設計、API設計）に基づく実装計画です。

## フェーズ1: 環境構築とベース準備
フロントエンド（React）、バックエンド（FastAPI）、データベース（PostgreSQL）の基本環境を構築します。

1. **データベース（PostgreSQL）環境構築**
   - ローカル開発用PostgreSQL DBの作成
   - 接続設定の整備
2. **バックエンド（FastAPI）環境構築**
   - Python仮想環境設定
   - FastAPI、SQLAlchemy、Alembic（マイグレーション用）、Pydantic等のライブラリ導入
   - プロジェクトディレクトリ構造の作成（`main.py`, `routers`, `models`, `schemas`, `crud` 等）
3. **フロントエンド（React）環境構築**
   - Vite等によるReactプロジェクト作成（TypeScript推奨）
   - UIコンポーネントライブラリ（Material-UI等が必要な場合）、ルーティング設定
   - ディレクトリ構造作成（`components`, `pages`, `api`, `types` 等）

## フェーズ2: データベース実装
`04_db_design.md` に基づき、テーブル定義と初期データを準備します。

1. **マイグレーション環境整備**
   - Alembicの初期化と設定
2. **モデル定義とテーブル作成**
   - プロジェクト (`projects`)、タスク (`tasks`)、サブタスク (`subtasks`) テーブル
   - マスタテーブル (`mst_subtask_types`, `mst_statuses`, `mst_members`, `mst_holidays`, `system_settings`)
   - 外部キー制約、チェック制約の設定
3. **初期データ・シード作成**
   - 初期ステータスマスタ（New, In Progress, Removed等）、テスト用データ

## フェーズ3: バックエンド（API）実装
`05_api_list_design.md` に基づき、各APIエンドポイントを実装します。

1. **マスタAPI・共通補助APIの実装**
   - マスタ系のCRUD（GET、POST、PATCH等）
   - 初期データ一括取得API (`/api/initial-data`)
2. **プロジェクト・タスク・サブタスク 基本CRUD APIの実装**
   - それぞれの登録、更新、論理削除、個別取得API
3. **メイン画面WBSデータ取得APIの実装**
   - フィルタリング条件に基づく、プロジェクト〜サブタスクの階層データ取得 (`/api/wbs`)
4. **特殊操作APIの実装**
   - 並び替え・移動API (`/reorder`, `/move`)
   - 複製API (`/copy`)
   - 自動入力補助計算API（日付・工数計算用）

## フェーズ4: フロントエンド実装（基本＆マスタ）
`02_ui_requirements.md` に基づき、画面とコンポーネントを構築します。

1. **共通UI・レイアウト作成**
   - ヘッダー、通知トースト領域、共通レイアウトコンポーネントの構築
2. **API連携・状態管理の準備**
   - AxiosやReact Query等を用いたAPIクライアント設定と状態管理基盤の整備
3. **マスタ管理画面の実装**
   - ステータス、担当者、種別などのマスタの閲覧・追加・編集UI
4. **共有コンポーネントの作成**
   - 確認ダイアログ（削除、Removed用）
   - 各種フォーム要素（日付カレンダー、プルダウン等）

## フェーズ5: フロントエンド実装（メイン画面・WBSツリー）
メイン画面の左半分にあたるWBSツリー領域を実装します。

1. **WBSツリー表示の基本実装**
   - 階層構造（プロジェクト＞タスク＞サブタスク）のレンダリング
   - 展開/折りたたみ機能
   - スライサー（フィルター）機能の実装
2. **インライン編集機能**
   - ツリー行での直接編集UI（名称、日付、ステータス、担当者等）と即時自動保存機能
   - オートコレクト・自動入力ボタンの動作
3. **詳細項目のUI実装**
   - サブタスク詳細表示ダイアログの実装（メモ、チケットID、予定・実績工数等）
4. **特殊操作（追加・削除・ドラッグ＆ドロップ）**
   - 新規行追加操作、論理削除
   - 並び替えと移動を実現するドラッグ＆ドロップ機能

## フェーズ6: フロントエンド実装（ガントチャート）
メイン画面の右半分にあたるガント領域と関連機能を実装します。

1. **ガントチャートのベース描画**
   - ヘッダー（日付・週表示）、今日線の描画
   - ツリー領域との縦スクロール同期と横スクロールの独立
2. **ガントバーのマッピングと描画**
   - サブタスクの計画・実績バー描画（ステータス色、遅延強調）
   - タスク・プロジェクトの集約バー描画
3. **リサイズ領域と連動機能の実装**
   - ツリーとガント間のリサイズバー実装
   - 表示期間（n週間）切り替えや、WBSツリー更新時の即時再判定と再描画の処理

## フェーズ7: 結合テストと仕上げ
全体の挙動を通しで確認し、品質を高めます。

1. **全体動作確認**
   - インライン編集時のガントチャートリアルタイム反映
   - 複数操作時の自動保存ステータス更新やエラーハンドリング
2. **不具合修正とパフォーマンス調整**
   - 大量データ時のレンダリング最適化（必要に応じて仮想スクロール等）
   - UIの微調整操作性の向上（Excelライクな操作感への調整）
